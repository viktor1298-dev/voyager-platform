    **Step 1 — Create apps/api/src/routes/metrics-stream.ts:**
    Follow the exact pattern from `ai-stream.ts` (raw `reply.raw.writeHead/write`). Implementation per 02-RESEARCH.md Pattern 1 + Pattern 4:

    1. Validate query params with Zod: `z.object({ clusterId: z.string().uuid() })`
    2. Authenticate via `auth.api.getSession({ headers })` (same pattern as ai-stream.ts lines 33-40)
    3. Verify cluster exists: `db.select({ id: clusters.id }).from(clusters).where(eq(clusters.id, clusterId))` — return 404 if not found
    4. Write SSE headers: `content-type: text/event-stream; charset=utf-8`, `cache-control: no-cache, no-transform`, `x-accel-buffering: no`, `connection: keep-alive`
    5. Generate connectionId via `crypto.randomUUID()`
    6. Create handler function: `const handler = (event: MetricsStreamEvent) => { reply.raw.write(\`event: metrics\ndata: ${JSON.stringify(event)}\n\n\`) }`
    7. Subscribe: `voyagerEmitter.on(\`metrics-stream:${clusterId}\`, handler)` then `metricsStreamJob.subscribe(clusterId, connectionId)`
    8. Start heartbeat: `setInterval(() => reply.raw.write(':keepalive\n\n'), SSE_HEARTBEAT_INTERVAL_MS)`
    9. Cleanup on `request.raw.on('close')`: clearInterval heartbeat, `voyagerEmitter.off(...)` with same handler ref, `metricsStreamJob.unsubscribe(clusterId, connectionId)`, `reply.raw.end()`
    10. Export `registerMetricsStreamRoute` as async function taking `FastifyInstance`
    11. Add connection limits: track active connections per cluster (max 10) and global (max 50). Return 429 if exceeded. Use a simple module-level Map for counters.
