**Step 3: TypeScript type check (per D-03)**

```bash
pnpm typecheck
```

If typecheck fails, analyze each error:
- **Missing imports**: A file imports something that was renamed or removed in the other branch. Fix the import path.
- **Type mismatches**: A function signature changed on one side while a call site is from the other. Update the call site or signature.
- **Missing exports**: A barrel file (index.ts) is missing an export that another package depends on. Add the export.
- **Duplicate identifiers**: Both branches added the same import or variable differently. Remove the duplicate.
- **`m` not found**: A Motion import was missed in Plan 02's normalization. Replace `m` with `motion`.

For EACH fix: edit the file, then `git add <file>` to include the fix in the merge commit.

**Step 4: Lint check (per D-03)**

```bash
pnpm lint
```

If lint fails:
- Biome formatting issues from manual conflict resolution edits. Run `pnpm lint --write` or manually fix the flagged lines.
- For each fix: `git add <file>`.

**Step 5: Full monorepo build (per D-03)**

```bash
pnpm build
```

If build fails, common post-merge issues (per PITFALLS.md):
- **Missing module**: `Cannot find module './node-metrics-history.js'` — the schema file was accidentally deleted. Restore from `pre-merge-snapshot` tag.
- **tRPC type errors**: Client-side references a procedure that does not exist on the server. Fix the router or the client usage.
- **Next.js SSR errors**: Missing environment variable references or broken dynamic imports. Fix the import or add the missing env var to the build config.
- **Barrel export issues**: A re-export barrel file is missing entries. Add the missing exports.

For EACH fix: edit the file, then `git add <file>`.

**Step 6: Unit tests (per D-03)**

```bash
pnpm test
```

If tests fail:
- Read the test failure output carefully. Determine if the failure is from:
  - A merge resolution error (wrong code in a file) — fix the source code
  - A test that needs updating (references old behavior) — fix the test
  - An import/type issue caught by the test runner — fix the import

For EACH fix: edit the file, then `git add <file>`.
