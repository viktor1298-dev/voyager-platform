**File 1: `apps/web/src/hooks/useMetricsSSE.ts`**

Create a hook that manages an EventSource connection to `/api/metrics/stream`:

```typescript
interface UseMetricsSSEOptions {
  enabled: boolean       // false = no connection
  maxPoints: number      // buffer capacity (25 for 5m, 65 for 15m)
  rangeMs: number        // time window for eviction
}

interface UseMetricsSSEReturn {
  points: MetricsDataPoint[]
  hasData: boolean
  connectionState: SSEConnectionState
  lastTimestamp: number | null
}
```

Implementation details:
- `'use client'` directive at top
- Create `MetricsBuffer` instance via `useRef` (not useState -- avoid re-renders on internal buffer mutations). When `maxPoints` or `rangeMs` changes, create a new buffer instance.
- On `enabled=true`: create `new EventSource(\`/api/metrics/stream?clusterId=\${clusterId}\`)` in a useEffect
- Listen for `message` events (the SSE endpoint sends generic message events with JSON data). Parse JSON as `MetricsStreamEvent`. Call `convertSSEEvent()` to map to `MetricsDataPoint`. Push into buffer. Call a state setter to trigger re-render with `buffer.toArray()`.
- Listen for `open` event -> set connectionState to `'connected'`, reset reconnect delay
- Listen for `error` event -> close EventSource, set connectionState to `'reconnecting'`, schedule reconnect with exponential backoff using `SSE_INITIAL_RECONNECT_DELAY_MS` (1s), `SSE_RECONNECT_BACKOFF_MULTIPLIER` (2x), `SSE_MAX_RECONNECT_DELAY_MS` (30s) from `@voyager/config/sse`
- **Reconnect logic (SSE-03):** Use a `reconnectTimeoutRef` to schedule `setTimeout(connect, delay)`. On each error, `delay = Math.min(delay * multiplier, maxDelay)`. On successful connection, reset delay to initial. Clear timeout on cleanup.
- **Visibility-aware lifecycle (SSE-04):** Add a `document.visibilitychange` listener:
  - `document.hidden === true` -> close EventSource, clear reconnect timeout, set state `'disconnected'`
  - `document.hidden === false` -> clear buffer (`bufferRef.current.clear()`), reconnect fresh, set state `'connecting'`
  - Guard with `typeof document !== 'undefined'` for SSR safety (Gotcha #13 from CLAUDE.md -- never branch on typeof window in render, but this is inside useEffect so it is safe)
- **Cleanup:** useEffect return function closes EventSource, clears reconnect timeout, removes visibilitychange listener
- When `enabled` changes to `false`: close EventSource, clear buffer, set state `'disconnected'`
- Track `lastTimestamp` from the most recent SSE point for "last updated" display

Import types from `@voyager/types` and constants from `@voyager/config/sse`. Import `MetricsBuffer` and `convertSSEEvent` from `../lib/metrics-buffer.js`. Import `MetricsDataPoint` from `../components/metrics/MetricsAreaChart.js`.

**File 2: `apps/web/src/hooks/useMetricsData.ts`**

Create a unified hook that abstracts SSE vs tRPC:

```typescript
type DataSourceMode = 'live' | 'historical'

interface UseMetricsDataReturn {
  data: MetricsDataPoint[]
  isLoading: boolean
  isError: boolean
  isLive: boolean
  connectionState: SSEConnectionState
  lastUpdated: number | null
  error: Error | null
  refetch: () => void
}

export function useMetricsData(clusterId: string, range: MetricsRange): UseMetricsDataReturn
```

Implementation (TIME-04):
- `'use client'` directive at top
- Determine mode: `const mode: DataSourceMode = (range === '5m' || range === '15m') ? 'live' : 'historical'`
- SSE parameters derived from range:
  - `5m`: maxPoints=25, rangeMs=5*60_000
  - `15m`: maxPoints=65, rangeMs=15*60_000
- Call `useMetricsSSE(clusterId, { enabled: mode === 'live', maxPoints, rangeMs })`
- Call `trpc.metrics.history.useQuery({ clusterId, range }, { enabled: mode === 'historical', staleTime: 30_000, retry: 2 })`
- Return unified shape:
  - `data`: if live, return SSE points; if historical, return `historyQuery.data?.data ?? []`
  - `isLoading`: if live, `!liveData.hasData && liveData.connectionState === 'connecting'`; if historical, `historyQuery.isLoading`
  - `isError`: if live, `liveData.connectionState === 'disconnected'` (after all retries); if historical, `historyQuery.isError`
  - `isLive`: `mode === 'live'`
  - `connectionState`: from SSE hook (pass through for live indicator)
  - `lastUpdated`: if live, `liveData.lastTimestamp`; if historical, `historyQuery.dataUpdatedAt || null`
  - `error`: if historical, `historyQuery.error`; if live, null
  - `refetch`: if historical, `historyQuery.refetch`; if live, no-op function
