# Phase 4 Plan 1: Dead Code Removal, DB Indexes, Build Honesty

## Objective

Remove ~900 lines of dead legacy watcher code, add missing database indexes on high-query tables, and remove `ignoreBuildErrors: true` from Next.js config to ensure build pipeline honesty.

## Context

- @apps/api/src/lib/cluster-watch-manager.ts (300 lines, replaced by watch-manager.ts)
- @apps/api/src/lib/resource-watch-manager.ts (309 lines, replaced by watch-manager.ts)
- @apps/api/src/lib/cluster-connection-state.ts (63 lines, only used by cluster-watch-manager)
- @apps/api/src/routers/subscriptions.ts (213 lines, unused tRPC subscription router)
- @apps/api/src/routers/index.ts (imports subscriptions router)
- @apps/web/next.config.ts (ignoreBuildErrors: true)
- @charts/voyager/sql/init.sql (missing indexes on events, nodes, audit_log, alert_history, health_history)

## Tasks

### Task 1: Remove dead legacy watcher code (CLEAN-01)
type="auto"

**Actions:**
1. Delete `apps/api/src/lib/cluster-watch-manager.ts`
2. Delete `apps/api/src/lib/resource-watch-manager.ts`
3. Delete `apps/api/src/lib/cluster-connection-state.ts`
4. Delete `apps/api/src/routers/subscriptions.ts`
5. Remove subscriptions router import and registration from `apps/api/src/routers/index.ts`
6. Update comment in `apps/api/src/lib/k8s-units.ts` that references cluster-watch-manager
7. Remove `stopAllWatchers` import from server.ts and the deprecated no-op function from k8s-watchers.ts
8. Remove the `streamLogs` and `watchDeploymentProgress` functions from k8s-watchers.ts that were only used by subscriptions router (if no other imports)

**Done when:**
- The four files no longer exist in the codebase
- No remaining imports reference the deleted files
- `pnpm --filter api build` succeeds (from main repo)

### Task 2: Add missing DB indexes (CLEAN-02)
type="auto"

**Actions:**
1. Add indexes to `charts/voyager/sql/init.sql` for:
   - `events`: index on (cluster_id, timestamp DESC) for time-range queries
   - `nodes`: index on (cluster_id) for per-cluster node listing
   - `audit_log`: index on (user_id, timestamp DESC) for user audit trail, and (timestamp DESC) for recent audit
   - `alert_history`: index on (alert_id, triggered_at DESC) for alert timeline
   - `health_history`: index on (cluster_id, checked_at DESC) for health timeline

**Done when:**
- All five tables have appropriate indexes in init.sql
- init.sql remains idempotent (CREATE INDEX IF NOT EXISTS)

### Task 3: Remove ignoreBuildErrors and verify clean build (CLEAN-03)
type="auto"

**Actions:**
1. Set `ignoreBuildErrors: false` in `apps/web/next.config.ts`
2. Verify `pnpm typecheck` passes with zero errors (from main repo)
3. If any TypeScript errors exist, fix them

**Done when:**
- `ignoreBuildErrors` is `false` in next.config.ts
- `pnpm typecheck` passes with 0 errors (verified in main repo)
