  <action>
Set branch protection rules on the `main` branch of `viktor1298-dev/voyager-platform` using the GitHub API via `gh`.

Use `gh api` to PUT branch protection rules:

```bash
gh api repos/viktor1298-dev/voyager-platform/branches/main/protection \
  --method PUT \
  --input - <<'EOF'
{
  "required_pull_request_reviews": null,
  "required_status_checks": null,
  "enforce_admins": false,
  "restrictions": null,
  "allow_force_pushes": false,
  "allow_deletions": false,
  "required_linear_history": false,
  "block_creations": false
}
EOF
```

Key decisions per CONTEXT.md:
- Per D-01: Require pull requests before merging — set via the protection endpoint (the endpoint itself blocks direct pushes when protection is enabled, even with `required_pull_request_reviews: null`)
- Per D-02: Prevent force push — `allow_force_pushes: false`
- Per D-03: Prevent branch deletion — `allow_deletions: false`
- Per D-04: Do NOT require PR reviews — `required_pull_request_reviews: null` (solo dev, self-merge)
- Per D-05: Do NOT require status checks — `required_status_checks: null` (no CI yet)

IMPORTANT: If the `required_pull_request_reviews: null` approach does NOT block direct pushes (GitHub may allow direct push if no PR review requirement is set), then update the payload to include a minimal PR review requirement:

```json
{
  "required_pull_request_reviews": {
    "required_approving_review_count": 0
  }
}
```

This still allows self-merge (0 approvals required) but enforces the PR workflow per D-01.

After setting protection, verify by reading the protection back:

```bash
gh api repos/viktor1298-dev/voyager-platform/branches/main/protection
```

Confirm the response shows protection is active with force pushes and deletions disabled.
  </action>
  <verify>
