New file `apps/web/src/components/metrics/DataFreshnessBadge.tsx`:

```typescript
interface DataFreshnessBadgeProps {
  dataUpdatedAt: number  // timestamp ms from historyQuery.dataUpdatedAt
  autoRefresh: boolean   // whether auto-refresh is active
}
```

Logic:
- Use `useState` + `useEffect` with a 5-second interval to recompute age from `Date.now() - dataUpdatedAt`.
- Three states based on age:
  - `dataUpdatedAt === 0` or age < 0: show nothing (no data yet)
  - `autoRefresh === true` AND age < 120_000 (2min): **"Live"** — green dot (pulsing) + text "Live". Use `var(--color-status-active)` (#00e599) for the dot/text color.
  - age < 300_000 (5min): **relative** — amber dot + text like "2m ago". Use `var(--color-status-warning)` (#f6c042). Calculate relative time: `<1m ago`, `1m ago`, `2m ago`, ..., `4m ago`.
  - age >= 300_000: **"Stale"** — red dot + text "Stale". Use `var(--color-status-error)` (#ff4d6a).
- Render: a `<span>` with `data-freshness` attribute set to `"live"`, `"recent"`, or `"stale"` for test hookability.
- Dot: 6x6px rounded-full inline-block with pulse animation (CSS `animate-pulse`) only in "Live" state.
- Font: `text-xs font-mono` to match the control bar style.
- Export: `export function DataFreshnessBadge(...)`.
