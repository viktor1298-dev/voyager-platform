# Phase 7 Plan 1: Chart Performance Optimization

## Objective

Optimize the metrics chart rendering pipeline so that 1000+ data points render without jank, crosshair synchronization doesn't cause render cascades, and ResponsiveContainer resizes are debounced.

## Context

- @apps/web/src/components/metrics/MetricsAreaChart.tsx ‚Äî current chart with `data.length` in key prop (causes full remount on data change)
- @apps/web/src/components/metrics/MetricsTimeSeriesPanel.tsx ‚Äî 4-panel layout with no crosshair synchronization
- @apps/api/src/routers/metrics.ts ‚Äî backend metrics router returning bucketed data (in-memory JS bucketing)
- @.planning/research/PITFALLS.md ‚Äî documents key performance pitfalls (key-prop remount, crosshair render cascade, ResponsiveContainer resize thrashing)
- @.planning/research/ARCHITECTURE.md ‚Äî recommended architecture for crosshair sync via shared state

## Tasks

### Task 1: LTTB Downsampling + Key Prop Fix

type="auto"

**Do:**
1. Create `apps/web/src/lib/lttb.ts` ‚Äî inline LTTB (Largest Triangle Three Buckets) downsampling algorithm (~50 LOC, no external dependency)
2. Fix `MetricsAreaChart.tsx` key prop: remove `data.length` from key ‚Äî use only `${range}-${activeMetrics.join('-')}` so data updates don't cause full SVG remount
3. Apply LTTB in `MetricsTimeSeriesPanel.tsx` ‚Äî when data exceeds 200 points, downsample to ~200 visual points before passing to each chart panel
4. Add `animationDuration={0}` alongside existing `isAnimationActive={false}` on all Area elements to eliminate the known 1500ms dot render delay

**Done when:**
- LTTB module exists with typed export
- Key prop no longer includes `data.length`
- Data arrays > 200 points are downsampled before chart render
- `animationDuration={0}` set on all Area elements

### Task 2: Crosshair Synchronization with Throttled Shared State

type="auto"

**Do:**
1. Create `apps/web/src/components/metrics/CrosshairProvider.tsx` ‚Äî React context that holds shared crosshair timestamp state, updated via `requestAnimationFrame`-throttled handler
2. Create `apps/web/src/components/metrics/CrosshairCursor.tsx` ‚Äî custom Recharts cursor component that renders a vertical line at the crosshair position, reading from CrosshairProvider context
3. Modify `MetricsAreaChart.tsx` ‚Äî accept `syncId` prop, use `onMouseMove`/`onMouseLeave` to update CrosshairProvider, render `CrosshairCursor` as custom cursor
4. Wrap the 4-panel grid in `MetricsTimeSeriesPanel.tsx` with `<CrosshairProvider>` so all panels share state
5. Memoize `CustomTooltip` with `React.memo` and ensure it receives primitive props

**Done when:**
- Hovering one panel shows crosshair line on all 4 panels at the same timestamp
- Mouse movement is throttled to ~16ms (RAF) ‚Äî no render cascade
- CustomTooltip is memoized

### Task 3: ResponsiveContainer Debounce + Build Verification

type="auto"

**Do:**
1. Create `apps/web/src/components/metrics/DebouncedResponsiveContainer.tsx` ‚Äî wrapper around Recharts `ResponsiveContainer` that debounces resize callbacks (150ms default) to prevent layout thrashing during window resize
2. Replace all `<ResponsiveContainer>` usage in `MetricsAreaChart.tsx` with the debounced version
3. Run `pnpm typecheck` and `pnpm build` to verify no type errors or build failures

**Done when:**
- DebouncedResponsiveContainer exists and is used in MetricsAreaChart
- `pnpm typecheck` passes with 0 errors
- `pnpm build` succeeds

## Verification

```bash
pnpm typecheck && pnpm build
```

## Success Criteria

1. LTTB downsampling limits chart data points to ~200 for any dataset size
2. Chart key prop doesn't cause full remount on data updates
3. Crosshair synchronization across 4 panels uses throttled shared state (not Recharts syncId)
4. ResponsiveContainer resizes are debounced
5. `pnpm typecheck` and `pnpm build` pass with 0 errors
