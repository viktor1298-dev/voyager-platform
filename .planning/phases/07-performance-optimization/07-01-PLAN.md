  <action>
    **Step 1 — Create `apps/web/src/lib/lttb.ts`:**

    Implement the LTTB algorithm as a single pure function (~50 LOC, no external dependencies). The function signature:

    ```typescript
    export function lttb(data: MetricsDataPoint[], targetPoints: number): MetricsDataPoint[]
    ```

    Import MetricsDataPoint from `@/components/metrics/MetricsAreaChart`.

    Algorithm (Largest Triangle Three Buckets):
    1. If `data.length <= targetPoints`, return `data` as-is (identity case).
    2. Always keep first and last points.
    3. Divide the middle data (index 1 to length-2) into `targetPoints - 2` equal-sized buckets.
    4. For each bucket, compute the triangle area formed by: (a) the previously selected point, (b) each candidate point in the current bucket, (c) the average point of the next bucket.
    5. Select the candidate with the largest triangle area (preserves visual extremes).
    6. Use the `timestamp` field (parsed to epoch ms via `new Date(ts).getTime()`) as the X coordinate and `cpu` (fallback 0 for null) as the Y coordinate for area calculation. CPU is a representative metric — the algorithm preserves temporal shape regardless of which metric the user views.

    **Step 2 — Create `apps/web/src/lib/lttb.test.ts`:**

    Use vitest (same patterns as `metrics-buffer.test.ts`). Create a `makePoint` helper that produces `MetricsDataPoint` objects with a given timestamp and cpu value. Tests:

    1. **Empty input**: `lttb([], 200)` returns `[]`
    2. **Below threshold**: 50-point array with target 200 returns all 50 points unchanged
    3. **Exact threshold**: 200-point array with target 200 returns all 200 points unchanged
    4. **Downsampling 1000 to 200**: Output length is exactly 200
    5. **First/last preserved**: Output[0] === input[0] and output[last] === input[last]
    6. **Order preserved**: Output timestamps are monotonically increasing
    7. **Extremes preserved**: Create input with a spike (cpu=95 at index 500 in a sea of cpu=50). Verify the spike point appears in the output.
    8. **Null values handled**: Points with `cpu: null` do not throw; function returns correct length
    9. **Edge case targetPoints <= 2**: `lttb(data, 1)` returns at least first and last points

    **Step 3 — Wire into `useMetricsData.ts`:**

    Add `import { lttb } from '@/lib/lttb'` at the top.

    Define a constant: `const LTTB_TARGET_POINTS = 200`

    Apply downsampling to both return paths:

    - **Live path** (line ~70): Change `data: liveData.points` to `data: lttb(liveData.points, LTTB_TARGET_POINTS)`. (Live data is typically <=65 points so lttb will be a no-op, but this future-proofs.)

    Wrap the lttb calls in `useMemo` to avoid recomputing on every render. The memo deps should be the raw data reference (e.g., `historyQuery.data` or `liveData.points`).

    Do NOT modify `MetricsTimeSeriesPanel.tsx` — it already calls `useMetricsData` via its own `historyQuery` path. The downsampling must happen inside `useMetricsData` so all consumers benefit.
