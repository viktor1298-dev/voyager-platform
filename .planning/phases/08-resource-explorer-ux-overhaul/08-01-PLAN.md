**ResourcePageScaffold.tsx** — Generic page wrapper (per D-01):
```typescript
'use client'
interface ResourcePageScaffoldProps<T> {
  title: string
  icon: ReactNode
  queryResult: { data: T[] | undefined; isLoading: boolean; error: unknown }
  getNamespace: (item: T) => string
  getKey: (item: T) => string
  filterFn: (item: T, query: string) => boolean
  renderSummary: (item: T) => ReactNode
  renderDetail: (item: T) => ReactNode
  searchPlaceholder?: string
  emptyMessage?: string
  emptyDescription?: string
  flatList?: boolean  // For namespaces page — skip namespace grouping
}
export function ResourcePageScaffold<T>({ ... }: ResourcePageScaffoldProps<T>) { ... }
```

ResourcePageScaffold implementation:
1. State: `searchQuery` (string), `expandAll` (boolean, default false)
2. Derive `filteredItems` via `useMemo` — filter with `filterFn` if searchQuery non-empty
3. Derive `grouped` via `useMemo` — `Map<string, T[]>` using `getNamespace`, sorted by namespace name. If `flatList=true`, put all items under a single group.
4. Render: SearchFilterBar at top → NamespaceGroup sections → ExpandableCard per item (controlled via `expanded={expandAll}`) with `renderSummary` and `renderDetail`
5. Loading state: 3 skeleton cards (use Skeleton from shadcn)
6. Empty state: icon + emptyMessage + emptyDescription centered
7. hasCredentials check: use `'hasCredentials' in (queryResult.data ?? {})` pattern is NOT needed — the scaffold receives pre-filtered queryResult. The cluster query + hasCredentials check stays in the page component that wraps the scaffold.

NOTE: The scaffold does NOT include any SSE/live-data code. Live data is handled at the layout level by Plan 02's useResourceSSE hook.
