**1. Create ResourcePageScaffold.tsx (per D-01, Research Pattern 1):**

Create `apps/web/src/components/resource/ResourcePageScaffold.tsx`:

```typescript
'use client'

import { useParams } from 'next/navigation'
import { getClusterIdFromRouteSegment } from '@/components/cluster-route'
import { ExpandableCard } from '@/components/expandable'
import { Skeleton } from '@/components/ui/skeleton'
import { trpc } from '@/lib/trpc'
import { usePageTitle } from '@/hooks/usePageTitle'
import type { ReactNode } from 'react'
import { useMemo, useState } from 'react'
import { NamespaceGroup } from './NamespaceGroup'
import { SearchFilterBar } from './SearchFilterBar'

interface ResourcePageScaffoldProps<T> {
  title: string
  icon?: ReactNode
  queryHook: (clusterId: string) => {
    data: T[] | undefined
    isLoading: boolean
    isError: boolean
    error: { message: string } | null
  }
  getNamespace: (item: T) => string
  getKey: (item: T) => string
  filterFn: (item: T, query: string) => boolean
  renderSummary: (item: T, expandAll: boolean) => ReactNode
  renderDetail: (item: T) => ReactNode
  searchPlaceholder: string
  emptyMessage: string
  emptyDescription?: string
  emptyIcon?: ReactNode
  flatList?: boolean  // For pages that don't group by namespace (e.g., Namespaces page)
}

export function ResourcePageScaffold<T>({ ... }: ResourcePageScaffoldProps<T>) { ... }
```

The scaffold encapsulates this pattern (extracted from pods/page.tsx):

a. **Cluster resolution boilerplate** (lines 366-382 of pods page):
   - `useParams()` to get route segment
   - `getClusterIdFromRouteSegment(routeSegment)` for clusterId
   - `trpc.clusters.get.useQuery({ id: clusterId })` for DB cluster
   - `resolvedId = dbCluster.data?.id ?? clusterId`
   - `hasCredentials = Boolean((dbCluster.data as Record<string, unknown> | undefined)?.hasCredentials)`
   - Call `usePageTitle(title)`

b. **Data query** (via the `queryHook` prop):
   - `const query = queryHook(resolvedId)` — enabled only when `hasCredentials && !!resolvedId`
   - Note: the queryHook is called by the scaffold, not the page. The page passes a function like `(clusterId) => trpc.services.listDetail.useQuery({ clusterId }, { enabled: hasCredentials && !!clusterId, refetchInterval: 30000 })`
   - IMPORTANT: Actually, since hooks can't be called conditionally, the scaffold should accept the query result directly instead. Revise: the page calls the hook and passes `data`, `isLoading`, `isError`, `error` as props. The scaffold renders based on these.

   **Revised interface — remove queryHook, accept data directly:**
   ```typescript
   interface ResourcePageScaffoldProps<T> {
     title: string
     icon?: ReactNode
     data: T[] | undefined
     isLoading: boolean
     hasCredentials: boolean
     getNamespace: (item: T) => string
     getKey: (item: T) => string
     filterFn: (item: T, query: string) => boolean
     renderSummary: (item: T) => ReactNode
     renderDetail: (item: T) => ReactNode
     searchPlaceholder: string
     emptyMessage: string
     emptyDescription?: string
     emptyIcon?: ReactNode
     flatList?: boolean
   }
   ```

c. **Search + filter state:**
   - `const [searchQuery, setSearchQuery] = useState('')`
   - `const [expandAll, setExpandAll] = useState(false)`
   - `const items = (data ?? []) as T[]`
   - Filter: `const filtered = useMemo(() => { if (!searchQuery.trim()) return items; const q = searchQuery.toLowerCase().trim(); return items.filter(item => filterFn(item, q)) }, [items, searchQuery, filterFn])`

d. **Namespace grouping:**
   - If `flatList` is false (default): group by namespace using the same `useMemo` + Map pattern from pods page
   - If `flatList` is true: render cards directly without namespace sections

e. **Rendering (top to bottom):**
   - `<h1 className="sr-only">{title}</h1>` (accessibility)
   - If `!hasCredentials`: render the "Live data unavailable" empty state (same as pods page line 421-434)
   - If `isLoading`: render `<Skeleton>` placeholders (3 rows)
   - Else render `<SearchFilterBar>` then:
     - If `filtered.length === 0`: empty state with `emptyMessage` / `emptyDescription` / `emptyIcon`
     - If `flatList`: render cards directly in `<div className="space-y-1">`
     - Else: render grouped namespace sections using `<NamespaceGroup>` containing `<ExpandableCard expanded={expandAll} onExpandedChange={() => setExpandAll(false)}>` for each item
   - Each `<ExpandableCard>` gets `key={getKey(item)}`, `summary={renderSummary(item)}`, `expanded={expandAll || undefined}` (pass `undefined` when not expanded-all so cards revert to uncontrolled mode)

   **IMPORTANT expand-all behavior:** When `expandAll` is true, ALL cards are controlled with `expanded={true}`. When user clicks any individual card to collapse it, set `expandAll` to false and let all cards return to uncontrolled mode (they'll keep their last state via `defaultExpanded`). Actually, simpler approach: maintain a `Set<string>` of expanded keys. "Expand All" adds all keys. "Collapse All" clears the set. Individual card clicks toggle their key. The scaffold passes `expanded={expandedKeys.has(key)}` and `onExpandedChange={(exp) => toggleKey(key, exp)}`.

   **Simplest correct approach:** Use `expandAll` as a "force" flag. When `expandAll` is null (default), cards are uncontrolled. When `expandAll` is true, pass `expanded={true}`. When `expandAll` is false, pass `expanded={false}`. After user clicks "Expand All", set to `true`. After "Collapse All", set to `false`. After any individual card click, reset to `null` (uncontrolled) so cards manage themselves. Type: `const [expandAll, setExpandAll] = useState<boolean | null>(null)`. Pass to ExpandableCard: `expanded={expandAll ?? undefined}`.
