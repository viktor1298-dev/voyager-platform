**Create `apps/api/src/routers/yaml.ts`** — universal resource YAML fetcher (per D-03):

- Export `yamlRouter` using `router({...})`
- Single procedure `get` using `protectedProcedure` (not admin — viewers need YAML access)
- Input schema: `z.object({ clusterId: z.string().uuid(), resourceType: z.string(), name: z.string(), namespace: z.string().optional() })`
- `resourceType` accepts: `pods`, `deployments`, `services`, `configmaps`, `secrets`, `statefulsets`, `daemonsets`, `jobs`, `cronjobs`, `ingresses`, `hpa`, `pvcs`, `namespaces`, `nodes`, `networkpolicies`, `resourcequotas`
- Implementation: Use `clusterClientPool.getClient(clusterId)` to get KubeConfig, then call appropriate K8s API based on resourceType. Use a `RESOURCE_API_MAP` lookup pattern (see RESEARCH.md Pattern 5).
- Cache with `CACHE_KEYS.k8sYaml(clusterId, resourceType, name, namespace)` at 15s TTL
- Error handling: `handleK8sError(err, 'get resource yaml')`
- Return the raw K8s API object as JSON (frontend converts to YAML)

**Add cache key** to `apps/api/src/lib/cache-keys.ts`:
```typescript
k8sYaml: (clusterId: string, resourceType: string, name: string, ns?: string) =>
  `k8s:yaml:${clusterId}:${resourceType}:${ns ?? 'cluster'}:${name}`,
```

**Register in `apps/api/src/routers/index.ts`:**
- Import: `import { yamlRouter } from './yaml.js'`
- Add to appRouter: `yaml: yamlRouter,`
