```typescript
interface ResourceDiffProps {
  clusterId: string
  resourceType: string
  resourceName: string
  namespace?: string
}
```

Implementation:
- Fetch current resource via `trpc.yaml.get.useQuery({ clusterId, resourceType, name: resourceName, namespace })`
- Extract `kubectl.kubernetes.io/last-applied-configuration` annotation from the fetched resource's `metadata.annotations`
- If annotation exists: parse it as JSON, convert both sides to YAML using `stringify` from `yaml` package
- Use `react-diff-viewer-continued` for rendering the diff:
  - Import `ReactDiffViewer` from `react-diff-viewer-continued`
  - Props: `oldValue={lastAppliedYaml}`, `newValue={currentYaml}`, `splitView={true}`
  - Custom styles using CSS custom properties: `--color-diff-added-bg`, `--color-diff-removed-bg`, `--color-diff-modified-bg`, `--color-diff-added-text`, `--color-diff-removed-text`
  - Left header: "Last Applied", Right header: "Current"
- Copy button on each panel header — copies that side's full YAML to clipboard
- If no `kubectl.kubernetes.io/last-applied-configuration` annotation: show centered message "No last-applied configuration found. Resource was not created with kubectl apply."
- If no diff detected: show centered message "No changes detected. Resource matches last-applied configuration."
- For Helm-managed resources (detect `app.kubernetes.io/managed-by: Helm` label): show toggle "Compare Helm Revisions" (placeholder for now — actual Helm revision diff will be wired in Plan 07)
- Font: 13px Geist Mono, line-height 1.4
- Synchronized scroll between panels
- Max-height: 500px with overflow-y-auto
