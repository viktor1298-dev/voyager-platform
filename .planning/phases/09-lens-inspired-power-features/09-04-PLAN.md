**StatefulSets router** — add 3 procedures:

1. `restart`: `adminProcedure`, input `{ clusterId, name, namespace }`, PATCH `metadata.annotations['kubectl.kubernetes.io/restartedAt']` to current ISO timestamp using `AppsV1Api.patchNamespacedStatefulSet`. Log audit `'statefulset.restart'`. Return `{ success: true, restartedAt }`.

2. `scale`: `adminProcedure`, input `{ clusterId, name, namespace, replicas: z.number().int().min(0).max(100) }`, PATCH `spec.replicas` using `AppsV1Api.patchNamespacedStatefulSet`. Log audit `'statefulset.scale'`. Return `{ success: true, replicas }`.

3. `delete`: `adminProcedure`, input `{ clusterId, name, namespace }`, call `AppsV1Api.deleteNamespacedStatefulSet`. Log audit `'statefulset.delete'`. Return `{ success: true }`.

**DaemonSets router** — add 2 procedures:

1. `restart`: `adminProcedure`, input `{ clusterId, name, namespace }`, PATCH `metadata.annotations['kubectl.kubernetes.io/restartedAt']` using `AppsV1Api.patchNamespacedDaemonSet`. Log audit `'daemonset.restart'`. Return `{ success: true, restartedAt }`.

2. `delete`: `adminProcedure`, input `{ clusterId, name, namespace }`, call `AppsV1Api.deleteNamespacedDaemonSet`. Log audit `'daemonset.delete'`. Return `{ success: true }`.

Note: DaemonSets do NOT have scale — they run one pod per node (per D-05).

**Pods router** — add 1 procedure:

1. `delete`: `adminProcedure`, input `{ clusterId, name, namespace }`, call `CoreV1Api.deleteNamespacedPod`. Log audit `'pod.delete'`. Return `{ success: true }`.

For ALL mutations:
- Use `clusterClientPool.getClient(clusterId)` to get KubeConfig

- Use `{ headers: { 'Content-Type': 'application/strategic-merge-patch+json' } }` for PATCH operations (same as deployments.ts)
- Use `logAudit(ctx, ...)` for audit logging (same pattern as deployments.ts)
