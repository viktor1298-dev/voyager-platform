**Create `apps/api/src/routes/log-stream.ts`** (per D-04, RESEARCH Pattern 2):

Export `async function registerLogStreamRoute(app: FastifyInstance): Promise<void>`

Route: `app.get('/api/logs/stream', async (request, reply) => { ... })`

Implementation following metrics-stream.ts SSE pattern:
1. **Authenticate** — Use `auth.api.getSession({ headers: request.headers })`. Return 401 if no session.
2. **Parse query params** — `clusterId` (uuid), `podName`, `namespace`, `container` (optional, defaults to first container), `tailLines` (optional, default 100)
3. **Validate** — Zod schema for params
4. **Verify cluster exists** — Query DB for cluster by ID, return 404 if not found
5. **Get K8s client** — `const kc = await clusterClientPool.getClient(clusterId)`
6. **SSE headers** — Same as metrics-stream:
   ```typescript
   reply.raw.writeHead(200, {
     'content-type': 'text/event-stream; charset=utf-8',
     'cache-control': 'no-cache, no-transform',
     'x-accel-buffering': 'no',
     connection: 'keep-alive',
   })
   ```
7. **Start log follow** — Use the `Log` class from `@kubernetes/client-node` (more reliable than `readNamespacedPodLog` for streaming — per RESEARCH open question):
   ```typescript
   import { Log } from '@kubernetes/client-node'
   const log = new Log(kc)
   const logStream = new PassThrough()
   await log.log(namespace, podName, container, logStream, {
     follow: true, tailLines, timestamps: true, pretty: false,
   })
   ```
8. **Pipe to SSE** — Listen on logStream data events, split by newlines, send each line as SSE event:
   ```typescript
   logStream.on('data', (chunk: Buffer) => {
     const lines = chunk.toString().split('\n').filter(Boolean)
     for (const line of lines) {
       reply.raw.write(`event: log\ndata: ${JSON.stringify({ line, timestamp: Date.now() })}\n\n`)
     }
   })
   ```
9. **Heartbeat** — Same as metrics-stream: send `:heartbeat\n\n` every 30s to keep connection alive
10. **Cleanup on disconnect** — `request.raw.on('close', () => { logStream.destroy(); clearInterval(heartbeat) })`
11. **Error handling** — logStream error event: send SSE error event, then close. Max line buffer: track line count, destroy stream after 10,000 lines with "max lines reached" event.

**Register in server.ts**:
- Import: `import { registerLogStreamRoute } from './routes/log-stream.js'`
- Call: `await registerLogStreamRoute(app)` alongside other route registrations

Run `pnpm typecheck` to verify.
