**Create `apps/api/src/routers/helm.ts`** (per D-07, RESEARCH Pattern 3):

Export `helmRouter` with procedures:

1. `list`: `protectedProcedure`, input `{ clusterId: z.string().uuid() }`
   - Get KubeConfig via `clusterClientPool.getClient(clusterId)`
   - List secrets with `coreV1.listSecretForAllNamespaces({ fieldSelector: 'type=helm.sh/release.v1' })`
   - BUT: only fetch metadata first (use `labelSelector: 'owner=helm'` to find Helm secrets by label, not field selector on type — fieldSelector on type may not work in all K8s versions. Fall back to filtering by `secret.type === 'helm.sh/release.v1'`)
   - Group by release name (from label `name`), keep only latest revision per release
   - For list view: return summary data WITHOUT decoding full release (Pitfall 3)
   - Summary: `{ name, namespace, chartName, chartVersion, appVersion, status, revision, updatedAt }` — extract from secret labels/annotations
   - Cache with `CACHE_KEYS.k8sHelmReleases(clusterId)` at 30s TTL

2. `get`: `protectedProcedure`, input `{ clusterId, releaseName: z.string(), namespace: z.string() }`
   - Find the latest revision secret for this release
   - **Decode Helm release** (per RESEARCH Pattern 3):
     ```typescript
     const releaseData = secret.data?.release
     const compressed = Buffer.from(releaseData, 'base64')
     const jsonBuffer = gunzipSync(compressed)  // from 'node:zlib'
     const release = JSON.parse(jsonBuffer.toString('utf-8'))
     ```
   - Return full release: `{ name, namespace, chartName, chartVersion, appVersion, status, revision, firstDeployed, lastDeployed, values (the config object), notes, resources (from manifest parsing) }`
   - Cache with `CACHE_KEYS.k8sHelmRelease(clusterId, releaseName, namespace)` at 30s TTL

3. `revisions`: `protectedProcedure`, input `{ clusterId, releaseName, namespace }`
   - List ALL secrets for this release (all revisions)
   - Return array of `{ revision, status, updatedAt, description }` sorted by revision desc
   - Cache with `CACHE_KEYS.k8sHelmRevisions(clusterId, releaseName, namespace)` at 30s TTL

**Create `apps/api/src/routers/crds.ts`** (per D-13, RESEARCH Pattern CRD):

Export `crdsRouter` with procedures:

1. `list`: `protectedProcedure`, input `{ clusterId }`
   - Get KubeConfig, create `ApiextensionsV1Api` client
   - Call `listCustomResourceDefinition()`
   - Map to `{ name (spec.names.plural), group (spec.group), version (spec.versions[0].name), scope (spec.scope), plural (spec.names.plural), kind (spec.names.kind), instanceCount }` — for instanceCount, set to 0 initially (fetched on expand)
   - Cache with `CACHE_KEYS.k8sCrds(clusterId)` at 30s TTL

2. `instances`: `protectedProcedure`, input `{ clusterId, group, version, plural, scope: z.enum(['Namespaced', 'Cluster']) }`
   - Use `CustomObjectsApi` (per RESEARCH Pattern CRD instance listing)
   - For Cluster scope: `listClusterCustomObject({ group, version, plural })`
   - For Namespaced scope: list across all namespaces (fetch namespace list, then list per namespace — per RESEARCH code)
   - Return array of `{ name, namespace, uid, createdAt }` (metadata only)
   - Cache with `CACHE_KEYS.k8sCrdInstances(clusterId, group, plural)` at 15s TTL

3. `instanceYaml`: `protectedProcedure`, input `{ clusterId, group, version, plural, name, namespace }` (optional namespace for cluster-scoped)
   - Fetch single custom object and return raw JSON for YAML viewer

**Add cache keys** to cache-keys.ts:

```typescript
k8sHelmReleases: (clusterId: string) => `k8s:helm:releases:${clusterId}`,

k8sCrds: (clusterId: string) => `k8s:crds:${clusterId}`,
k8sCrdInstances: (clusterId: string, group: string, plural: string) => `k8s:crds:instances:${clusterId}:${group}:${plural}`,
```
