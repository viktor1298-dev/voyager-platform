   - Fetch all 4 RBAC resource types in parallel (per RESEARCH code):
     ```typescript
     const rbacApi = kc.makeApiClient(k8s.RbacAuthorizationV1Api)
     const [clusterRoles, clusterRoleBindings, roles, roleBindings] = await Promise.all([
       rbacApi.listClusterRole(),
       rbacApi.listClusterRoleBinding(),
       rbacApi.listRoleForAllNamespaces(),
       rbacApi.listRoleBindingForAllNamespaces(),
     ])
     ```
   - Build matrix: Map each binding to its role, extract rules (verbs x resources), build subject->resource->verbs map
   - Filter out `system:*` subjects by default (per Pitfall 8)
   - If `namespace` provided, also include namespace-scoped Roles/RoleBindings
   - Return: `{ subjects: string[], resources: string[], matrix: Record<string, Record<string, string[]>> }` where matrix[subject][resource] = ['get','list','create','update','delete'] (the verbs)
   - Cache with `CACHE_KEYS.k8sRbac(clusterId)` at 60s TTL (per Pitfall 8 — heavy computation)

2. `bindingDetail`: `protectedProcedure`, input `{ clusterId, subject, resource }`
   - Return the full binding chain for a specific subject+resource intersection
   - Return: `{ bindings: { roleName, bindingName, namespace?, verbs }[] }`
