## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, compact badge internal padding |
| sm | 8px | Compact element spacing, DetailTab gaps, toolbar button gaps |
| md | 16px | Default element spacing, card padding, panel padding |
| lg | 24px | Section padding, ExpandableCard body padding |
| xl | 32px | Layout gaps, GroupedTabBar row spacing |
| 2xl | 48px | Major section breaks between feature areas |
| 3xl | 64px | Page-level spacing (reserved, unlikely in Phase 9) |

Exceptions:
- Terminal (xterm.js) internal padding: 12px (xterm.js option `theme.padding`, not layout token)
- Drag handle hit zone: 12px height (resizable panel divider, enough for mouse+touch)
- RBAC matrix cell: 36px x 36px minimum (CRUD letter grid readability)
- React Flow node minimum: 48px height (graph node touch target)

Source: Existing patterns in globals.css (`px-4`, `p-4`, `gap-1.5`, `py-2.5` in DetailTabs)

## Interaction Contracts

### I-01: Terminal Drawer (D-02)

| Behavior | Specification |
|----------|---------------|
| Trigger | "Exec" button in pod detail panel ActionToolbar |
| Presentation | Bottom drawer, slides up from viewport bottom |
| Initial height | 40% of viewport height |
| Min height | 200px |
| Max height | 80% of viewport height |
| Resize | Drag handle at top of drawer, cursor `ns-resize`, 12px hit zone |
| Multi-terminal | Tabbed interface above terminal area. Each pod exec = new tab |
| Tab content | Pod name + container name (truncated at 24 chars) |
| Tab close | X icon on each tab, removes terminal session |
| Session persistence | Stays open during in-page navigation. Manual close only |
| Container selection | Dropdown in terminal header when pod has 2+ containers. Default: first container |
| Enter animation | `slideUp` from y: 100% to y: 0, spring stiffness 350, damping 24 |
| Exit animation | Reverse slideUp, duration 150ms |
| Keyboard | Ctrl+` toggles drawer visibility (VS Code convention) |
| Collapsed state | Thin 32px bar at bottom showing session count, click to expand |
| Focal point | Active terminal session text cursor -- the blinking cursor in the xterm.js viewport draws the eye to the input area where the user types commands |

| Behavior | Specification |
|----------|---------------|
| Location | "[YAML]" tab in every ExpandableCard DetailTabs |
| Content | Full resource JSON converted to YAML, syntax-highlighted |

### I-03: Resource Diff (D-14)

|----------|---------------|

| Action | Confirmation | Details |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus management | Terminal drawer traps focus when open. Esc key closes drawer and returns focus to trigger button |
| Keyboard navigation | RBAC matrix cells navigable with arrow keys. Graph nodes focusable with Tab |
| Screen reader | YAML viewer uses `role="log"` with `aria-label="Resource YAML"`. Diff uses `aria-label="Resource diff"` |
| Reduced motion | All new Motion components check `useReducedMotion()`. Graph animations skip to final state |
| Color alone | RBAC cells show CRUD letters, not just color. Status badges include text, not just color dot |
| Focus rings | 2px solid `--color-accent`, 2px offset on all interactive elements (existing global style) |
| Terminal a11y | xterm.js `screenReaderMode: true` option enabled |
