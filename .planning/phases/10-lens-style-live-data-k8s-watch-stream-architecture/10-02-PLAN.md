<objective>
Rewrite the SSE resource-stream endpoint to carry full transformed resource objects (D-01) with 1-second server-side batching (D-02). The current endpoint sends signal-only events (`{resourceType, changeType, name, namespace}`) that force the client to refetch. The new endpoint sends complete data (`{type: 'ADDED', resourceType, object: <transformed>}`) so the client can apply directly via setQueryData.

Purpose: Eliminate all refetch round-trips. When a K8s Watch event fires, the transformed resource object arrives at the browser via SSE within 0-1 seconds.

Output: Rewritten resource-stream.ts, unit tests for SSE format and batching.
</objective>

           events: batch,
           timestamp: new Date().toISOString(),

         reply.raw.write(`event: watch\ndata: ${JSON.stringify(payload)}\n\n`)
