<objective>
Migrate all 15 watched resource tRPC routers + topology + clusters to read from WatchManager's in-memory store (D-07) instead of calling the K8s API via cached(). When watches are active, data comes from informer cache (~0ms). When not active, fall back to direct K8s API call.

Purpose: Eliminate Redis cache layer and K8s API round-trips for watched resources. The WatchManager's informer cache is always current (updated by watch events in real-time), so routers get instant reads.

Output: 17 modified routers reading from WatchManager with K8s API fallback.
</objective>
