Output: Modified resource-stream.ts (immediate write + snapshot), compression-disabled metrics/log stream routes, updated SSE config constant.

       (Add `{ config: { compress: false } }` as second argument to `app.get`)

    2. **Remove the batch buffer entirely** — Delete the `batch` array, `batchTimer`, and `flushBatch()` function. Delete the `RESOURCE_STREAM_BUFFER_MS` import (it will remain in config at value 0 for documentation).

    3. **Remove the initial load window** — Delete the `INITIAL_LOAD_WINDOW_MS` constant (line ~32), the `initialLoadWindow` variable, the `initialLoadTimer` setTimeout, and the `if (initialLoadWindow && event.type === 'ADDED') return` guard in `onWatchEvent`.

    4. **Add RESOURCE_DEFS import** — Import the RESOURCE_DEFS array from watch-manager. Since RESOURCE_DEFS is currently a module-level `const` (not exported), you need to export it. Add `export` before `const RESOURCE_DEFS` in watch-manager.ts. Then import it in resource-stream.ts:
       ```typescript
       import { watchManager, RESOURCE_DEFS } from '../lib/watch-manager.js'
       ```

    5. **Add snapshot event after subscribe** — After `await watchManager.subscribe(clusterId)`, iterate over RESOURCE_DEFS and send a `snapshot` event per resource type with the current informer cache:
       ```typescript
       // Send snapshot — current informer state for each resource type
       for (const def of RESOURCE_DEFS) {
         const resources = watchManager.getResources(clusterId, def.type)
         if (resources && resources.length > 0) {
           const mapped = resources.map((obj) => def.mapper(obj, clusterId))
           try {
             reply.raw.write(
               `event: snapshot\ndata: ${JSON.stringify({ resourceType: def.type, items: mapped })}\n\n`
             )
           } catch { /* connection closed */ }
         }
       }
       ```

    6. **Rewrite onWatchEvent to immediate write** — Replace the batch-based handler with:
       ```typescript
       const onWatchEvent = (event: WatchEvent): void => {
         const payload: WatchEventBatch = {
           clusterId,
           events: [event],
           timestamp: new Date().toISOString(),
         }
         try {
           reply.raw.write(`event: watch\ndata: ${JSON.stringify(payload)}\n\n`)
         } catch { /* connection closed */ }
       }
       ```

    7. **Update cleanup** — In the `request.raw.on('close', ...)` handler, remove `clearTimeout(initialLoadTimer)` and the `batchTimer` cleanup (both deleted). Keep heartbeat cleanup, event listener removal, unsubscribe, and connection decrement.

    8. **Update packages/config/src/sse.ts** — Change `RESOURCE_STREAM_BUFFER_MS` from `1_000` to `0` and update the comment:
       ```typescript
       /** Resource stream: buffer window (ms). Set to 0 — Phase 11 uses immediate flush */
       export const RESOURCE_STREAM_BUFFER_MS = 0
       ```

    9. **Export RESOURCE_DEFS from watch-manager.ts** — Add `export` keyword before `const RESOURCE_DEFS: ResourceDef[]` (line ~78 of watch-manager.ts). Also export the `ResourceDef` interface.
