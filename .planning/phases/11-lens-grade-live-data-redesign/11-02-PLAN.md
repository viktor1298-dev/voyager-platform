<objective>
Create the Zustand resource store and useResources hook that will replace TanStack Query's setQueryData for live K8s data. This is the new reactive data layer for all 15 watched resource types.

Purpose: TanStack Query's setQueryData is designed for request/response, not push channels. A dedicated Zustand store with selector-based subscriptions prevents unnecessary re-renders (only components watching the changed resource type re-render) and eliminates stale closure risks.

Output: resource-store.ts (Zustand store), useResources.ts (consumer hook), resource-store.test.ts (unit tests).
</objective>

  <action>
    **Create apps/web/src/stores/resource-store.ts:**

    ```typescript
    import { create } from 'zustand'
    import { subscribeWithSelector } from 'zustand/middleware'
    import type { ResourceType, WatchEvent } from '@voyager/types'
    ```

    Store state interface:
    - `resources: Map<string, unknown[]>` — keyed by `${clusterId}:${resourceType}`
    - `connectionState: Record<string, 'initializing' | 'connected' | 'reconnecting' | 'disconnected'>`

    Actions:
    - `setResources(clusterId, type, items)` — replaces entire array for key (used by snapshot event)
    - `applyEvent(clusterId, event: WatchEvent)` — applies ADDED/MODIFIED/DELETED to the array for `event.resourceType`
    - `setConnectionState(clusterId, state)` — updates connection state for a cluster
    - `clearCluster(clusterId)` — removes all keys starting with `${clusterId}:` from resources Map, sets connectionState to 'disconnected'

    Key implementation details for `applyEvent`:
    - Cast `event.object` to `{ name: string; namespace?: string | null }`
    - ADDED: findIndex by name+namespace. If found, replace (upsert). If not found, append.
    - MODIFIED: map over array, replace matching item. (Keep array as-is if no match found.)
    - DELETED: filter out matching item.
    - Always create a new Map reference (`new Map(state.resources)`) so React detects the change.

    Export as `useResourceStore`.

    Use `subscribeWithSelector` middleware so consumers can subscribe to specific slices.

    **Create apps/web/src/stores/__tests__/resource-store.test.ts:**

    Use Vitest. Import `useResourceStore` directly. Use `useResourceStore.getState()` and `useResourceStore.setState()` for testing (no React rendering needed). Reset store between tests with `useResourceStore.setState({ resources: new Map(), connectionState: {} })`.

    Write tests for all 9 behaviors listed above.

    Create `apps/web/src/hooks/useResources.ts` with two hooks:

    1. **useClusterResources<T>(clusterId: string, type: ResourceType): T[]**
       - Reads from Zustand store using a memoized selector: `useResourceStore(useCallback((s) => (s.resources.get(\`${clusterId}:${type}\`) ?? []) as T[], [clusterId, type]))`
       - Returns typed array of resources for the given cluster + type
       - Returns empty array if no data exists

    2. **useConnectionState(clusterId: string): ConnectionState**
       - Reads `connectionState[clusterId]` from the Zustand store
       - Returns `'disconnected'` as default if clusterId not in store
       - Export the `ConnectionState` type (same as current: `'initializing' | 'connected' | 'reconnecting' | 'disconnected'`)
