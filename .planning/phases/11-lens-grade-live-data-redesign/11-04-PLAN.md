<objective>
Migrate all 15 resource pages from tRPC useQuery to Zustand useClusterResources. The SSE snapshot provides initial data on connect, so tRPC list queries are no longer needed for watched resources. Keep tRPC for non-watched data (Helm, CRDs, RBAC, etc).

Purpose: Complete the architecture migration. With snapshot-on-connect providing initial data and live watch events pushing updates, the tRPC list queries for watched resources are redundant. Removing them eliminates unnecessary API round-trips and the last remnants of the polling-era architecture.

Output: 15+ updated page files reading from Zustand instead of tRPC, plus cleanup of stale code.
</objective>

  <action>
    For each resource page, apply this migration pattern:

    **Step 1 — Add import:**
    ```typescript
    import { useClusterResources, useConnectionState } from '@/hooks/useResources'
    ```

    **Step 2 — Replace tRPC query with Zustand read:**

    Before (example — pods):
    ```typescript
    const podsQuery = trpc.pods.list.useQuery({ clusterId: resolvedId }, { enabled: hasCredentials })
    const pods = podsQuery.data ?? []
    const isLoading = podsQuery.isLoading
    ```

    After:
    ```typescript
    const pods = useClusterResources<PodData>(resolvedId, 'pods')
    const connectionState = useConnectionState(resolvedId)
    const isLoading = pods.length === 0 && connectionState === 'initializing'
    ```

    Where `PodData` is the existing TypeScript type/interface for a pod item (inferred from the current page — look at how `podsQuery.data` items are typed or destructured).

    **Step 3 — Fix isLoading/isError/isFetching references:**
    - `isLoading`: `pods.length === 0 && connectionState === 'initializing'`
    - `isError`: `connectionState === 'disconnected' && pods.length === 0`
    - `isFetching`/`isRefetching`: Remove — SSE is always live
    - Remove any `refetchInterval` since data comes via SSE now

    **Step 4 — Remove tRPC import if it was the last usage:**
    If the page has no other tRPC queries left (check for mutations, other non-watched queries), remove the `trpc` import. However, many pages also use tRPC for mutations (restart, scale, delete) — keep those.

    **Page-specific notes:**

    **pods/page.tsx:** Replace `trpc.pods.list.useQuery`. Keep any mutations (restart, delete). The pods type is already defined inline or via the router return type — use the shape `{ name: string; namespace: string; status: string; ... }`. If no explicit interface exists, use `Record<string, unknown> & { name: string; namespace?: string | null }` or define a lightweight `PodItem` type at the top.

    **deployments/page.tsx:** Replace `trpc.deployments.listDetail.useQuery` (note: `listDetail`, not `list`). Keep restart/scale mutations.

    **services/page.tsx:** Replace `trpc.services.list.useQuery` AND `trpc.services.listDetail.useQuery` (both exist in useResourceSSE's setQueryData). Use the same data from Zustand for both.

    **nodes/page.tsx:** Replace `trpc.nodes.listLive.useQuery`. The page currently has a `dbNodes` query that is used when NOT live — since we're always live now, we can simplify: just use `useClusterResources<NodeData>(resolvedId, 'nodes')`. Remove the `isLive` / `effectiveIsLive` branching if it only served to choose between live vs DB data. Keep `trpc.nodes.list.useQuery` ONLY if it's used for the DB fallback case; otherwise, remove it too.

    **events/page.tsx:** This page has DUAL data sources — live K8s events + DB events. Replace the live K8s events query with Zustand. Keep the DB events query (`trpc.events.list.useQuery` with `source: 'db'` or similar).

    **page.tsx (overview):** Uses `dbNodes` and `dbEvents` for overview. Replace the live data queries with Zustand reads. Keep anomalies query (non-watched DB data).

    **IMPORTANT: Do NOT change the shape of data passed to child components.** The Zustand store contains the exact same data shape as the tRPC response (both use the same mappers on the backend). Components consuming the data need zero changes.
