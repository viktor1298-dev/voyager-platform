A

Fluid Full-Width (Recommended)

Remove the max-width cap entirely. Content flows to fill available space at all resolutions. Dashboard grid, tables, and cards naturally expand. Add an ultra-wide breakpoint (1920px+) to the dashboard grid so widgets can use the extra columns.

Pros

  • Uses 100% of screen real estate
  • Simplest change (~5 files)
  • Dashboard grid already responsive via react-grid-layout
  • Tables and lists benefit the most

Cons

  • Very long text lines on ultra-wide (but dashboards are mostly cards/tables, not prose)
  • Need to add xl/2xl breakpoint to DashboardGrid
B

Increased Cap + Center

Raise max-width to 2200px and add mx-auto to center. Content expands significantly but doesn't stretch infinitely on 4K/5K screens. Centered layout looks intentional rather than broken.

Pros

  • Content doesn't stretch too thin on 5K+ screens
  • Centered layout looks polished
  • Still gains ~800px on your ultra-wide

Cons

  • Still caps — users with even wider screens hit the same problem
  • Arbitrary magic number (why 2200 and not 2400?)
C

Hybrid: Full-Width + Per-Section Prose Cap

Remove the outer max-width. Add max-width only to text-heavy elements (descriptions, headers) while letting grids, tables, and cards fill the full width. Best of both worlds.

Pros

  • Grids and tables use all available space
  • Text remains readable (capped at ~80ch)
  • Most professional approach used by Grafana, Datadog, etc.

Cons

  • More files to touch (every page's prose sections)
  • Slightly more complex than A
